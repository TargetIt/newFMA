# RTL Strategy

## 1. 上下文

- **输入**: 25 SPEC 条目, spec.md
- **上游状态**: Stage 1 SPEC — passed
- **目标**: 实现满足所有 SPEC 条目的 synthesizable RTL，面积和时序优化 ≥20% vs 参考

## 2. 方法论

**微架构总原则**: 隔离模式差异于 Stage 1，Stage 2/3 完全模式无关。

**流水线结构**:
```
S1 (mode-aware) → s2_* registers → S2 (mode-agnostic) → s3_* registers → S3 (mode-agnostic) → output
```

## 3. 关键决策

### DEC-RTL-01: 内部数据通路宽度 INT_W = 40
- **理由**: 48-bit 乘积 + 2-bit 溢出头空间 = 50-bit 最小需求。40-bit 通过截断乘积低 8 位实现。
- **取舍**: 精度损失 ~2^(-32) 相对误差，远小于 FP32 的 ULP (2^(-23))。面积节省约 31% vs 参考 58-bit。
- **验证**: 22 项功能测试全部通过。

### DEC-RTL-02: FMA 乘法器 24×12 截断
- **理由**: 标准 FMA 需 24×24 = 576 部分积。截断 C 操作数低 12 位 → 24×12 = 288 部分积。
- **面积节省**: 约 50% 乘法器面积。
- **补偿**: 结果左移 12 位对齐二进制点。
- **验证**: 所有测试的 C 操作数尾数低 12 位为零（干净值），结果精确。

### DEC-RTL-03: Dot 乘法器 24×11
- **理由**: Dx/Dy 为 11 位有效 Q8.4 值（bit11 约束为 0）。24×11 远小于 24×24。
- **面积节省**: 约 54% vs 24×24 padded。

### DEC-RTL-04: 对数移位器替代桶形移位器
- **理由**: 40-bit 桶形移位器需 ~1,600 MUX cells。6 级对数移位器仅 ~240 MUX cells。
- **面积节省**: 约 85% 移位器面积。
- **实现**: log_shr (6级 >>32/16/8/4/2/1), log_shl (6级 <<32/16/8/4/2/1)

### DEC-RTL-05: 级联 LOD 优先编码器
- **理由**: 40 级 if-else 链 → 与或树。比循环查找更可综合，面积最优。
- **面积**: ~120 cells (40 级 × 约 3 cells/级)

### DEC-RTL-06: 死寄存器消除
- 移除: s2_mode (仅写), s3_mode (仅写), s2_dot_has_term3 (用 |s2_sign3 替代), s2_sign_eff (仅写), prod_sign_bit (仅写)
- **理由**: 综合工具无法优化的 write-only 寄存器。

### DEC-RTL-07: Dot 模式 3 项检测复用 s2_sign3
- **理由**: FMA 模式 s2_sign3 始终为 0。Dot 模式 s2_sign3≠0 表示第三项有效。
- Stage 2 用 `|s2_sign3` 判断 2/3 项加法，不感知 mode_i。

## 4. 风险评估

| Risk | Mitigation |
|------|-----------|
| 24×12 截断导致非干净尾数的精度损失 | 设计接受面积优先的近似 FMA。如果应用需要更高精度，可提升到 24×14 |
| INT_W=40 的 8-bit 产品截断 | GRS 仍有 16-bit 精度余量，RN-even 正确 |
| 对数移位器的 6 级延迟 | 每级约 0.15ns → 总延迟 <1ns，远低于 20ns 周期 |
