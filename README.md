# newFMA — FP32 FMA / Dot-Product RTL (SKY130 HD)

A 3-stage pipelined, area-optimized **approximate** FP32 Fused Multiply-Add
(`Y = A + B·C`) and Dot-Product (`Y = Ps + Px·Dx + Py·Dy`) unit, targeting the
SkyWater SKY130 HD standard-cell library. Open-source EDA flow (Icarus Verilog +
Yosys + ABC).

> **Approximate by design.** The 24×24 mantissa multiply is truncated to 24×12
> (12 LSBs of `C` dropped) and the internal datapath is narrowed to `INT_W=40`.
> This trades a small amount of numerical precision for ~25% area. Rounding is
> RN-even; inputs/outputs are flush-to-zero (FTZ). See `runs/final-v1/spec/spec.md`
> and the human-review gates in `runs/final-v1/goal.json`.

## Directory layout

```
rtl/   fma_fp32_dot3.v          Design (single module, 3-stage pipeline)
tb/    tb_fma_fp32_dot3.v       Testbench: 22 directed + model-driven regression
       ref_model.py              Bit-accurate Python reference model + vector generator
       test_vectors.hex          Golden vectors (regenerated by `make vectors`)
syn/   synthesis.ys             Yosys synthesis script template (@SKY130LIB@)
       constraints.sdc           OpenSTA constraints (20 ns clock)
       run_sta.py                Lightweight pessimistic STA cross-check
       area_report.txt           Latest synthesis area report
       sta_logs/sta_*ps.log      Multi-period ABC STA sweep (authoritative timing)
runs/  final-v1/                Goal/spec/logs/report for the official run
       opt-v1/                   Earlier optimization experiment
```

## Prerequisites

- **Icarus Verilog** (`iverilog`/`vvp`) and **Yosys** (with ABC) — e.g. from
  [oss-cad-suite](https://github.com/YosysHQ/oss-cad-suite-build).
- **SKY130HD** liberty file (`sky130_fd_sc_hd__tt_025C_1v80.lib`), e.g. via
  [volare](https://github.com/efabless/volare).

The Makefile defaults point at `~/tools/oss-cad-suite` and a volare PDK path;
override them for your machine:

```bash
make sim OSS_CAD=/path/to/oss-cad-suite
make sta SKY130LIB=/path/to/sky130_fd_sc_hd__tt_025C_1v80.lib
```

## Quick start

```bash
make lint      # iverilog -Wall compile check (0 warnings)
make sim       # generate golden vectors + run 54 tests (22 directed + 32 model-driven)
make synth     # synthesize to SKY130 HD (SKY130LIB, CLOCK_PERIOD=20000 by default)
make sta       # multi-period ABC STA sweep (8/10/12/16/20 ns)
make sta-tool  # lightweight pessimistic STA cross-check (NOT signoff)
make clean
```

## Verification

The bit-accurate Python model (`tb/ref_model.py`) mirrors the RTL datapath
**exactly** — including the 24×12 truncated multiply, `INT_W=40` width, log
shifters, 2-/3-term CPA, LOD, RN-even rounding and FTZ — so it is a genuine
golden model, not just a vector generator. It emits `tb/test_vectors.hex`
(32 vectors: the 22 directed cases + 10 **dirty-mantissa** cases that exercise
the truncation/sticky paths the directed suite skipped). The testbench loads
this file via `$readmemh` and checks every vector through the DUT.

**Result: 54 pass, 0 fail** (22 directed + 32 model-driven).

## Synthesis & timing (current INT_W=40 RTL, SKY130 HD tt_025C_1v80)

| Metric                       | Value                          |
|------------------------------|--------------------------------|
| Standard cells (mapped)      | 9,050                          |
| Chip area                    | 62,147 µm²                     |
| Sequential area              | 7,382 µm² (11.88%)             |
| Flip-flops                   | 295                            |
| Timing closure               | **8 ns (125 MHz)**, 0% area penalty |

Timing evidence: the ABC NLDM multi-period sweep (`syn/sta_logs/sta_8000ps.log`
… `sta_20000ps.log`) shows **identical cell count and area at every period from
8 ns to 20 ns** — i.e. the design closes at 8 ns with no upsizing. This is the
authoritative result; `syn/run_sta.py` provides only a pessimistic worst-case-sum
upper bound (not signoff).

## Reproducibility

Every artifact is regenerable from source via the Makefile. The committed
`syn/area_report.txt` and `syn/sta_logs/` reflect the currently tracked RTL.
