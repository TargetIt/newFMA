## 第二层：3 级流水线——时间展开

```
  valid_i ──► ┌──────────┐    ┌──────────┐    ┌──────────┐ ──► valid_o
  inputs ──►  │  STAGE 1 │───►│  STAGE 2 │───►│  STAGE 3 │ ──► y_o
              │  (组合)  │    │  (组合)  │    │  (组合)  │
              └────┬─────┘    └────┬─────┘    └────┬─────┘
                   │               │               │
              ┌────┴─────┐    ┌────┴─────┐    ┌────┴─────┐
              │ s2_* reg │    │ s3_* reg │    │ valid_o  │
              │ (296 FF) │    │ (~150 FF)│    │ y_o (32) │
              └──────────┘    └──────────┘    └──────────┘
                 ↗               ↗               ↗
            时钟沿锁存       时钟沿锁存        时钟沿锁存
```

> 每一级之间用寄存器（D Flip-Flop）隔开。
> 总共约 296 个 FF：S1→S2 有 ~150 个，S2→S3 有 ~110 个，输出有 33 个。

## 第三层：Stage 1 内部——拆包、乘、对齐

```
                           ┌──────────────────────────────────┐
  a_i ──► unpack_ftz() ───┤                                  │
  b_i ──► unpack_ftz() ───┤     ┌─────────────────────┐      │
  c_i ──► unpack_ftz() ───┤     │  Special Detect     │      │
                           │     │  NaN? Inf? Zero?    │      │
  unpack = {is_nan, is_inf,│     │  Inf×0? Inf取消?    │──────┼──► s2_special
            is_zero, sign, │     └─────────────────────┘      │    s2_special_result
            exp[7:0],      │                                   │
            mant[23:0]}     │     ┌─────────────────────┐      │
                           ├─────┤  FMA Multiplier      │      │
                           │     │  24×12 = 384 PPs     │      │
  mode_i ──► 0: b×c       │     │  prod = (B×C[23:12]) │      │
             1: px×dx+    │     │        << 12          │      │
                py×dy      │     └──────────┬──────────┘      │
                           │                │                  │
                           │     ┌──────────┴──────────┐      │
                           │     │  Normalize Product  │      │
                           │     │  if prod[47]: >> 1   │      │
                           │     │  else: keep [46:0]   │      │
                           │     └──────────┬──────────┘      │
                           │                │                  │
  40-bit data path ────────┤     ┌──────────┴──────────┐      │
                           │     │  Alignment          │      │
                           │     │  anchor = max(exp)   │      │
                           │     │  diff = exp diff     │      │
                           │     │  log_shr(small, diff)│      │
                           │     └──────────┬──────────┘      │
                           │                │                  │
                           └────────────────┼──────────────────┘
                                            ▼
                              To s2_term1, s2_term2, s2_term3
                              s2_sign1, s2_sign2, s2_sign3
                              s2_exp (anchor)
```

**Stage 1 的四个子步骤：**

```
Unpack ────────► Multiply ────────► Normalize ────────► Align
(拆包+FTZ)      (24×12 乘法)      (规范化乘积)        (统一指数)
   │                │                  │                  │
   ▼                ▼                  ▼                  ▼
分离符号/指数   面积优化乘法器    确保乘积在[1,2)    对数移位器对齐
/尾数, FTZ冲洗 截断12 LSB      范围, 调整乘积指数  两个操作数的小数点
```

## 第四层：Stage 2 内部——加法与前导一检测

```
  s2_term1 ──► ┌──────────────┐
  s2_sign1 ──► │ Signed Term  │──► t1 (41-bit signed)
               │ Conversion   │
  s2_term2 ──► │ 0→0, 1→+mag │──► t2
  s2_sign2 ──► │ 2→-mag      │──► t3
  s2_term3 ──► │              │
  s2_sign3 ──► └──────┬───────┘
                      │
              ┌───────┴───────┐
              │  CPA (加法器) │  41-bit Carry-Propagate Adder
              │  sum = t1+t2  │  (FMA: 2-term)
              │  (or t1+t2+t3)│  (Dot: 3-term, CSA+CPA)
              └───────┬───────┘
                      │
              ┌───────┴───────┐
              │  Abs + Sign   │  sum[40] → result_sign
              │  2's complement│  positive: pass through
              │               │  negative: ~sum + 1
              └───────┬───────┘
                      │
              ┌───────┴───────┐
              │  LOD (40-bit) │  Leading-One Detector
              │  Cascaded     │  级联优先编码器
              │  if-else      │  bit39→lod=0
              │  chain        │  bit38→lod=1
              │               │  ...bit0→lod=39
              └───────┬───────┘
                      │
                      ▼
          To s3: s3_mant (40-bit)
                 s3_lod (6-bit)
                 s3_exp (8-bit)
                 s3_result_sign (1-bit)
```

**LOD 的工作原理（级联优先编码器）：**

```
假如 sum_abs = 40'b 0000_0000_0000_0000_0000_1111_0000_0000_0000_0000
                                                ↑ MSB at bit 19

搜索过程（从高位到低位）：
  bit 39 = 0?  pass
  bit 38 = 0?  pass
  ...
  bit 20 = 0?  pass
  bit 19 = 1?  YES! → lod = 39 - 19 = 20

40 个 if-else 条目串成级联链，硬件上只是一个简单的与-或树
（and-or tree），每个条目 :
  if (sum_abs[N]) lod = 39 - N;
```

## 第五层：Stage 3 内部——归一化、舍入、打包

```
  s3_mant[39:0] ──► ┌──────────────────────┐
  s3_lod[5:0] ────► │  log_shl(6-stage)    │  对数左移器
                     │  stage1: <<32 if sa5 │  (与 Stage1 的 log_shr
                     │  stage2: <<16 if sa4 │   对称)
                     │  ...                 │
                     │  stage6: <<1  if sa0 │
                     └──────────┬───────────┘
                                │ shifted_mant[39:0]
                                │
              ┌─────────────────┴─────────────────┐
              │  Extract:                          │
              │  norm_mant = shifted[39:16] (24b) │
              │  guard     = shifted[15]           │
              │  round     = shifted[14]           │
              │  sticky    = |shifted[13:0]         │
              └─────────────────┬─────────────────┘
                                │
              ┌─────────────────┴─────────────────┐
              │  RN-even Round:                    │
              │  if (guard & (round|sticky|lsb))   │
              │    norm_mant += 1                  │
              │  if overflow (bit 23 set)          │
              │    norm_mant >>= 1; exp += 1        │
              └─────────────────┬─────────────────┘
                                │
              ┌─────────────────┴─────────────────┐
              │  norm_exp = anchor_exp + 1 - lod  │
              │  + rounding overflow adjustment    │
              │                                    │
              │  Output Checks:                    │
              │  exp <= 0?    → FTZ → 0x00000000  │
              │  exp >= 255?  → Overflow → Inf     │
              └─────────────────┬─────────────────┘
                                │
                                ▼
              y_o = {result_sign, norm_exp[7:0], norm_mant[22:0]}
              valid_o = s3_valid
```

**40 位内部寄存器的位布局：**

```
bit  39 38 ... 17 16 │ 15 │ 14 │ 13 ......... 0
     \____mantissa____/  \_guard_/  \___sticky___/
     hidden(39) +        (G) (R)      (S)
     23 fraction(38..16)
```

## 第六层：关键子模块剖视

### 6.1 对数移位器 (Logarithmic Shifter)

传统"桶形移位器"用 N×W 个 MUX（N=移位量，W=数据宽度）。对数移位器只需 log₂(N) × W 个。

```
输入 data[39:0], shamt[5:0]
         │
    ┌────┴────┐
    │ >> 32?  │  if shamt[5]: data >>= 32
    └────┬────┘
         │
    ┌────┴────┐
    │ >> 16?  │  if shamt[4]: data >>= 16
    └────┬────┘
         │
    ┌────┴────┐
    │ >> 8?   │  if shamt[3]: data >>= 8
    └────┬────┘
         │
    ┌────┴────┐
    │ >> 4?   │  if shamt[2]: data >>= 4
    └────┬────┘
         │
    ┌────┴────┐
    │ >> 2?   │  if shamt[1]: data >>= 2
    └────┬────┘
         │
    ┌────┴────┐
    │ >> 1?   │  if shamt[0]: data >>= 1
    └────┬────┘
         │
    输出 shifted[39:0]

面积节约: 桶形 ~2,304 cells → 对数 ~240 cells (6×40 MUX)
```

### 6.2 FMA 面积优化乘法器 (24×12)

```
标准 FMA: 24 × 24 位 → 576 部分积 → 大 Wallace 树
优化版:   24 × 12 位 → 288 部分积 → 小一半的 Wallace 树

    B_mant[23:0]          B 保留全部 24 位
×   C_mant[23:12]         C 只用高 12 位（截去低 12 位）
───────────────
    36-bit product        乘积只有 36 位有效
    << 12                  左移 12 位补偿（对齐二进制点）
───────────────
    48-bit aligned        填充到标准 48 位乘积位置

精度损失: 最低 12 位 C 尾数丢失 → 最多 2^(-35) 相对误差
面积节约: ~50% (576 → 288 部分积)
```

### 6.3 特殊值检测的优先级链

```
  ┌─────────┐
  │ 输入是否有│──YES──► y_o = 0x7FC00000 (quiet NaN)
  │ NaN？   │
  └────┬────┘
       │ NO
  ┌────┴────┐
  │ Inf × 0?│──YES──► y_o = 0x7FC00000 (NaN)
  └────┬────┘
       │ NO
  ┌────┴────┐
  │ Inf 抵消?│──YES──► y_o = 0x7FC00000 (NaN)
  │(Inf-Inf)│
  └────┬────┘
       │ NO
  ┌────┴────┐
  │ 有 Inf?  │──YES──► y_o = {sign, 0xFF, 0} (±Inf)
  └────┬────┘
       │ NO
       ▼
    正常计算路径
```

## 面积分布（SKY130 HD, 9,047 cells）

```
  xnor/xor (加法器+比较器):  ████████████████  31%  (~2,800 cells)
  nand/nor (随机逻辑):       ████████████      24%  (~2,200 cells)
  FFs (流水线寄存器):        ██████            12%  (~1,100 cells)*
  mux (移位器+LOD+控制):     ████████████████  30%  (~2,700 cells)
  其他 (inv, buf, etc.):    ██                 3%  (~250 cells)
                             ─────────────────────
                             合计:            9,047 cells
```

> *FF cell count 不同于 RTL 的 reg 数量。一个 multi-bit reg 会映射为多个 FF 物理单元。

## 总结：看懂 RTL 设计的三个层次

```
层次 1 (数学):    Y = A + B×C   或   Y = Ps + Px×Dx + Py×Dy
                  FP32 格式，IEEE 754 语义
                  
层次 2 (算法):    拆包 → 乘 → 对齐 → 加 → LOD → 归一化 → 舍入 → 打包
                 3 级流水线，RN-even，FTZ，特殊值优先级

层次 3 (硬件):    40 位数据通路，24×12 乘法器，对数移位器
                 级联 LOD，CPA 加法器，9,047 SKY130 cells
                 闭合 125 MHz，面积 ~32K µm²
```

把这三层串起来，就能从数学公式一路理解到硅片上的晶体管。
