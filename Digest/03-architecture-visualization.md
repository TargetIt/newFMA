# 第三章：FMA/Dot 融合架构 —— 哪里复用，哪里分叉

## 核心设计思想：隔离差异，统一后端

整个设计的灵魂就一句话：

> **把 FMA 和 Dot 的差异全部关在 Stage 1 里。Stage 2 和 Stage 3 不知道也不关心 mode_i。**

这就是为什么叫"融合"（fused）——不是两个独立的电路拼在一起，而是一条统一的流水线，只在入口处根据模式选择不同的计算路径。

---

## 第一层：融合总览 —— 分叉与汇合

```
                    mode_i
                      │
         ┌────────────┴────────────┐
         ▼                         ▼
    ┌─────────┐              ┌──────────┐
    │ FMA 路径│              │ Dot 路径  │
    │         │              │          │
    │ unpack  │              │ unpack   │
    │  ×3     │              │  ×3(*)   │
    │ 1个乘法 │              │ 2个乘法  │
    │ 2项对齐 │              │ 3项对齐  │
    │ 特殊值  │              │ 特殊值   │
    └────┬────┘              └────┬─────┘
         │         ┌─────────────┘
         ▼         ▼
    ┌─────────────────────────────┐
    │   统一的 s2_* 接口          │
    │   s2_term1, s2_term2,       │
    │   s2_term3, s2_sign1/2/3,   │
    │   s2_exp, s2_special        │
    └─────────────┬───────────────┘
                  │
                  ▼
    ┌─────────────────────────────┐
    │   Stage 2: 统一处理         │
    │   2项或3项加法 (|s2_sign3)  │
    │   CPA → Abs → LOD           │
    │   模式无关！                │
    └─────────────┬───────────────┘
                  │
                  ▼
    ┌─────────────────────────────┐
    │   Stage 3: 统一处理         │
    │   归一化 → RN-even → 打包   │
    │   完全模式无关！            │
    └─────────────┬───────────────┘
                  │
                  ▼
               y_o[31:0]
```

> (*) Dot 的 Px/Py 拆包用 `unpack_dot`，因为隐藏位由 `dot_p_msb_i` 指定而非自动置 1。
> Ps 仍用 `unpack_ftz`——和 FMA 的 A 完全一样。

## 第二层：Stage 1 内部 —— 隔离差异的地方

```
  a_i,b_i,c_i ──► ┌─────────────────────────────────────────┐
                  │           Stage 1: 分叉                  │
                  │                                          │
  mode_i=0 ──────►│  ┌──────────────────┐                   │
  (FMA)           │  │ unpack_ftz ×3    │                   │
                  │  │ B×C: 24×12 乘法器│                   │
                  │  │ resolve_special  │                   │
                  │  │ 2项对齐          │                   │
                  │  └────────┬─────────┘                   │
                  │           │                              │
  mode_i=1 ──────►│  ┌────────┴─────────┐                   │
  (Dot)           │  │ Ps: unpack_ftz   │                   │
  dx_i,dy_i,     │  │ Px,Py: unpack_dot│                   │
  dot_p_msb_i ──►│  │ Px×Dx: 24×11     │                   │
                  │  │ Py×Dy: 24×11     │                   │
                  │  │ LOD规范化 ×2     │                   │
                  │  │ 独立 Inf×0 检查  │                   │
                  │  │ 3项独立对齐      │                   │
                  │  └────────┬─────────┘                   │
                  │           │                              │
                  │           ▼                              │
                  │  ┌──────────────────┐                   │
                  │  │ 统一输出:         │                   │
                  │  │ s2_term1/2/3     │   ← 不管FMA还是Dot│
                  │  │ s2_sign1/2/3     │     这里都一样     │
                  │  │ s2_exp (锚点)    │                   │
                  │  │ s2_special       │                   │
                  │  └──────────────────┘                   │
                  └─────────────────────────────────────────┘
```

### FMA 独有的逻辑

```
乘法器: 24×12 截断 → 面积省 50%
  B_mant[23:0] × C_mant[23:12] << 12

规范化: 简单检查 prod_mant[47]
  if (bit47==1): >>1, exp+1  (乘积在[2,4), 右移1位)
  else: 保持                  (乘积在[1,2))

特殊值: resolve_special(B,C耦合)
  Inf×0 检查: (B_inf && C_zero) || (B_zero && C_inf)
```

### Dot 独有的逻辑

```
乘法器: 24×11 ×2 → 面积省 54% (vs 24×24 padded)
  Px_mant[23:0] × dx_i[10:0]
  Py_mant[23:0] × dy_i[10:0]

规范化: LOD 查找 MSB 位置, 变长移位到 bit46
  msb_pos = find_first_1(prod[46:0])
  shift = 46 - msb_pos
  prod_adj = prod << shift

特殊值: 独立 Inf×0 检查 (不能复用 resolve_special!)
  px_infzero = Px_inf && Dx==0  ← Px×Dx 的 Inf×0
  py_infzero = Py_inf && Dy==0  ← Py×Dy 的 Inf×0
  (因为 B×C 的耦合关系不适用于 Px×Dx + Py×Dy)

对齐: 三个操作数各自独立对齐到 max(exp_ps, exp_dx, exp_dy)
```

### 两模式共享的逻辑

```
unpack_ftz: FMA的A 和 Dot的Ps 用同一个函数
log_shr:    对齐移位 (两模式都用)
log_shl:    规范化移位 (Dot 的 LOD 之后)
{1'b0, mant, {AD_PAD{1'b0}}}: 40位寄存器位布局 (两模式相同)
anchor_exp: max(exponents) 的概念 (两模式相同)
```

## 第三层：Stage 2 —— 模式无关的统一加法器

```
  s2_term1 ──► ┌──────────────┐
  s2_sign1 ──► │ 符号→补码    │──► t1
  s2_term2 ──► │              │──► t2
  s2_sign2 ──► │ 0→0         │──► t3 (仅当 |s2_sign3)
  s2_term3 ──► │ 1→+原值     │
  s2_sign3 ──► │ 2→-原值     │
               └──────┬───────┘
                      │
          ┌───────────┴───────────┐
          │ if (|s2_sign3):       │  ← 唯一的模式感知！
          │   t1 + t2 + t3 (Dot) │
          │ else:                 │
          │   t1 + t2     (FMA)  │
          └───────────┬───────────┘
                      │
          ┌───────────┴───────────┐
          │ 取绝对值 + 符号       │
          │ sum[40] → result_sign │
          │ 正: 直通  负: ~x+1   │
          └───────────┬───────────┘
                      │
          ┌───────────┴───────────┐
          │ LOD: 级联优先编码器    │
          │ 从 bit39 向下扫 40 级 │
          │ 找第一个 1 → lod      │
          └───────────┬───────────┘
                      │
                      ▼
        s3_mant, s3_lod, s3_exp, s3_result_sign
```

> **关键设计决策：** Stage 2 用 `|s2_sign3`（第三项的符号是否非零）来判断是 2 项还是 3 项加法，而不是用 `mode_i`。这意味着 **Stage 2 不需要知道 mode_i**——它只看数据的形状。FMA 模式下 s2_sign3 永远为 0，自动走 2 项路径。

## 第四层：Stage 3 —— 完全模式无关

```
  s3_mant[39:0] ──► log_shl(×lod) ──► shifted[39:0]
  s3_lod[5:0] ────┘

  shifted[39:0]:
    norm_mant = [39:16]  (24位: 1隐藏 + 23小数)
    guard     = [15]
    round     = [14]
    sticky    = |[13:0]

  RN-even:
    if (guard & (round|sticky|norm_mant[0]))
      norm_mant += 1

  norm_exp = s3_exp + 1 - lod

  Output check:
    exp ≤ 0?    → FTZ → 0x00000000
    exp ≥ 255?  → Overflow → ±Inf
    else        → {sign, exp[7:0], mant[22:0]}
```

> Stage 3 是**零模式感知**的。它接收 s3_mant、s3_lod、s3_exp、s3_result_sign，输出一个 FP32 数。这些信号在 FMA 和 Dot 下含义完全相同。归一化、舍入、FTZ、溢出——**一套逻辑，两种模式共用。**

---

## 第五层：40 位内部寄存器的统一布局

```
bit  39 │ 38 ......... 17  16 │ 15 │ 14 │ 13 ......... 0
       │ \____mantissa_____/  │ G  │ R  │ \__sticky___/
  前导零   隐藏位(39) +
           23位小数(38..16)
```

FMA 和 Dot 在这个寄存器里的数据完全同构：
- 都是 `{1'b0, mantissa, trailing_zeros}` 格式
- 二进制点都在 bit 38 之后
- 锚点指数都在 s2_exp 里
- GRS 提取位置完全一致

## 第六层：融合收益 —— 为什么这样做省面积

```
如果 FMA 和 Dot 各自独立实现:
  FMA 数据通路  ~15,000 cells
  Dot 数据通路  ~12,000 cells
  ─────────────────────────
  合计          ~27,000 cells

融合实现 (实际):
  Stage 1: FMA 路径 + Dot 路径  ~5,500 cells
  Stage 2: 统一加法器            ~2,000 cells
  Stage 3: 统一归一化            ~1,500 cells
  ─────────────────────────
  合计                           ~9,000 cells

融合节省: ~67% (27,000 → 9,000)
```

节省来自：
1. **Stage 2 复用**: 加法器/绝对值/LOD 两模式共用，只多了一个 3 项加法的判断
2. **Stage 3 完全复用**: 归一化/舍入/打包零额外开销
3. **寄存器复用**: s2_*/s3_* 寄存器两模式共用，不增加 FF
4. **共享函数**: log_shr, log_shl, unpack_ftz, encode_special

## 完整数据流：标注复用

```
  a_i,b_i,c_i,dx_i,dy_i,dot_p_msb_i, mode_i
              │
    ┌─────────┴──────────┐
    │   Stage 1          │
    │                    │
    │  ┌─ FMA 路径 ─┐   │  ← 模式感知
    │  │ unpack_ftz │   │
    │  │ 24×12 mul  │   │
    │  │ 2项对齐    │   │
    │  └────────────┘   │
    │  ┌─ Dot 路径 ─┐   │  ← 模式感知
    │  │ unpack_dot │   │
    │  │ 24×11 ×2  │   │
    │  │ LOD规范化 │   │
    │  │ 3项对齐    │   │
    │  └────────────┘   │
    │                    │
    │  共享: log_shr     │  ← 复用
    │  共享: {1'b0,pad}  │  ← 复用
    │  共享: anchor_exp  │  ← 复用
    └────────┬───────────┘
             │ s2_* (统一接口)
    ┌────────┴───────────┐
    │   Stage 2          │
    │   2项/3项 CPA      │  ← 复用 (|s2_sign3 判断)

    │   Stage 3          │
    │   log_shl 归一化   │  ← 复用 (零模式感知)
    │   RN-even 舍入     │  ← 复用
    │   FTZ/Inf/Pack     │  ← 复用
    └────────┬───────────┘
             │
          y_o[31:0]
```

## 一句话总结

**FMA 和 Dot 的差异是"怎么把数对齐到公共指数"——这发生在 Stage 1。一旦对齐好，后面的加法、归一化、舍入、打包是同一套硬件，Stages 2/3 不知道也不关心 mode_i。**
