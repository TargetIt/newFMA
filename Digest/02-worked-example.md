# 第二章：一个例子贯穿始终 —— 跟踪 1.5 + 2.0 × 3.0

## 为什么选这个例子

`1.5 + 2.0 × 3.0 = 7.5` 这个例子恰到好处：
- 数值不复杂，心算都能验证
- 指数不完全相同（需要对齐）
- 乘积和加数都不是 "干净" 的 1.0（有尾数位）
- 能展示几乎所有关键路径

我们跟着 RTL 的 **3 级流水线**走一遍。

---

## 准备：看懂十六进制表示

在 RTL 仿真里，我们不说 "1.5"，而是说 `0x3FC00000`。先复习一下怎么转换：

```
1.5 的 FP32 表示：
1.5 = 1.1 (二进制) = 1.1 × 2^0
S = 0
E = 0 + 127 = 01111111 (二进制) = 0x7F
M = 10000000000000000000000 (二进制) = 0x400000

组装：S(1) + E(8) + M(23) = 0 01111111 10000000000000000000000
→ 位模式: 0011_1111_1100_0000_0000_0000_0000_0000
→ 十六进制: 0x3FC00000
```

同理：
```
2.0 → 0x40000000
3.0 → 0x40400000
7.5 → 0x40F00000
```

---

## Stage 1 (第 1 拍)：拆包、特殊值检测、乘法、对齐

### Step 1.1：拆包 (Unpack)

把 32-bit 输入拆成 {符号, 指数, 尾数}：

```
A (1.5): 0x3FC00000 = 0_01111111_10000000000000000000000
  → A_sign = 0, A_exp = 127, A_mant = 1.5 = 24'hC00000

B (2.0): 0x40000000 = 0_10000000_00000000000000000000000
  → B_sign = 0, B_exp = 128, B_mant = 1.0 = 24'h800000

C (3.0): 0x40400000 = 0_10000000_10000000000000000000000
  → C_sign = 0, C_exp = 128, C_mant = 1.5 = 24'hC00000
```

> 尾数是 24 位：1 位"隐藏的 1" + 23 位小数。如 1.5 = 1.1(二) = `1100_0000...`(24位) = `0xC00000`。

### Step 1.2：FTZ 检查

所有指数都不是 0 → 没有次正规数 → 不需要 flush。

### Step 1.3：特殊值检测

没有 NaN（exp≠255 或 mant=0），没有 Inf（exp≠255） → 走正常路径。

### Step 1.4：乘法

```
乘积尾数 = B_mant × C_mant
         = 24'h800000 × 24'hC00000
         = 1.0 × 1.5
         = 1.5
         = 48'h600000000000
                                         ┌── bit 47 = 0
         = 0110_0000... (48 bits)        │┌─ bit 46 = 1
                                         ││
乘积指数 = B_exp + C_exp - 127 = 128 + 128 - 127 = 129
乘积符号 = B_sign ^ C_sign = 0 ^ 0 = 0
```

> 乘积尾数是 48 位（两个 24 位相乘）。bit 47 = 0, bit 46 = 1 → 乘积在 [1, 2) 范围内，不需要规范化移位。

### Step 1.5：规范化乘积

```
prod_mant[47] = 0 → 已经在 [1,2) 范围
prod_mant_adj = prod_mant[46:0] = 47 位，MSB 在 bit 46
prod_exp_adj  = prod_exp = 129
```

### Step 1.6：对齐（Alignment）

A 的指数 = 127，乘积指数 = 129。差值 = 2。A 比较小，需要右移。

```
anchor_exp = 129 (取大的)

乘积：直接放入 40 位寄存器（高位对齐）
  prod_aligned = {1'b0, prod_mant_adj[46:8]}  ← 截去低 8 位
               = 40'h0800000000  （MSB 在位 35 = bit 46-8-3? ...）

加数 A：右移 2 位（缩小 4 倍）
  A_aligned = log_shr({1'b0, A_mant, 15'b0}, 2)
            = A_mant 右移 2 位 = 1.5 / 4 = 0.375
```

直观理解：
```
乘积:    1.5    (指数 129，作为锚点)
加数 A:  0.375  (本来是 1.5，但因为指数只有 127，被缩小了 4 倍)
```

### Step 1.7：打包，送入 Stage 2

以下是**实际仿真抓取的 s2 寄存器值**：

```
s2_valid   ← 1
s2_special ← 0  (正常路径)
s2_exp     ← 129
s2_term1   ← 40'h1800000000  (加数 A，已对齐 → 0.375)
s2_term2   ← 40'h6000000000  (乘积 → 1.5)
s2_term3   ← 40'h0000000000  (FMA 模式没有第三项)
s2_sign1   ← 1 (正)  s2_sign2 ← 1 (正)  s2_sign3 ← 0 (无)
```

> 这些数值来自 Icarus Verilog 仿真实测，见 `tb/tb_digest_verify.v`。
>
> `0x1800000000` 在 40-bit 寄存器中，bit[39]=0（前导零），bit[38]=0，bit[37]=0，
> bit[36]=1，bit[35]=1。二进制点在第 38 位之后，所以值 = 2^(-2)+2^(-3) = 0.25+0.125 = 0.375。
>
> `0x6000000000`：bit[38]=1，bit[37]=1 → 2^0+2^(-1) = 1.5。乘积尾数在锚点指数 129 下。<｜end▁of▁thinking｜>### Step 2.1：CPA（进位传播加法）

FMA 模式只有 2 项，直接加法。以下是**仿真实测的 S2 结果**：

```
s3_mant = 40'h7800000000  (绝对值和)
s3_lod  = 1              (前导一检测：MSB 在 bit 38)
s3_exp  = 129            (锚点指数，透传自 s2_exp)
```

验算：
```
  s2_term1 = 40'h1800000000  (0.375)
+ s2_term2 = 40'h6000000000  (1.5)
─────────────────────────
  sum      = 40'h7800000000  (1.875)
```

`0x7800000000` 位分析：bit[38]=1, bit[37]=1, bit[36]=1, bit[35]=1 → 1 + 0.5 + 0.25 + 0.125 = 1.875。 sum 为正 → result_sign = 0。

### Step 2.2：LOD（前导一检测）

```
bit 39 = 0  ← 跳过
bit 38 = 1  ← 找到了！→ lod = 39 - 38 = 1
```

`lod = 1` 意味着需要左移 1 位把 MSB 推到 bit 39。

### Step 2.3：打包，送入 Stage 3

```
s3_valid       ← 1
s3_special     ← 0
s3_exp         ← 129
s3_mant        ← 40'h7800000000 (1.875)
s3_lod         ← 1
s3_result_sign ← 0
s3_result_is_zero ← 0
```

---

## Stage 2 (第 2 拍)：加法、前导一检测

### Step 2.1：CPA（进位传播加法）

FMA 模式只有 2 项，直接加法：
```
sum = +0.375 + 1.5 = 1.875
```

在 40 位内部表示中：
```
  0.375  = 40'h0600000000  (bit 位对应...)
+ 1.5    = 40'h1800000000
─────────────────────────
= 1.875  = 40'h1E00000000
```

sum 是正数 → result_sign = 0

### Step 2.2：取绝对值和 LOD

sum 为正 → sum_abs = sum = 1.875

LOD（前导一检测）：找最高位的 1。
```
sum_abs 的 bit 位： ...00011110...
                       ↑ MSB 在位 36
lod = 39 - 36 = 3
```

> LOD 用级联优先编码器：从高位开始逐个检查 "这一位是 1 吗？"

### Step 2.3：打包，送入 Stage 3

```
s3_valid       ← 1
s3_special     ← 0
s3_exp         ← 129
s3_mant        ← 1.875 (绝对值)
s3_lod         ← 3
s3_result_sign ← 0
s3_result_is_zero ← 0
```

---

## Stage 3 (第 3 拍)：归一化、舍入、打包输出

### Step 3.1：归一化（左移 lod=1 位）

```
shifted = s3_mant << 1 = 40'h7800000000 << 1 = 40'hF000000000
```

移位后：bit[39]=1, bit[38]=1, bit[37]=1, bit[36]=1。MSB 在 bit 39。

### Step 3.2：提取尾数和 GRS 位

40 位内部表示中：
```
norm_mant[23:0] = shifted[39:16]  → 24'hF00000
  bit 23 (隐藏位) = 1
  bits 22:0 (小数) = 111_0000_0000_0000_0000_0000

guard  = shifted[15] = 0
round  = shifted[14] = 0
sticky = |shifted[13:0] = 0
```

### Step 3.3：归一化指数

```
norm_exp_9 = s3_exp + 1 - lod = 129 + 1 - 1 = 129
```

> 输出 FP32 的指数 = 129。验证：1.875 × 2^(129-127) = 1.875 × 4 = 7.5 ✓

### Step 3.4：RN-even 舍入

guard=0, round=0, sticky=0 → 恰好没有多余的位 → 不需要舍入。

### Step 3.5：打包输出

```
y_o = {result_sign, norm_exp[7:0], norm_mant[22:0]}
    = {0, 8'h81, 23'h700000}
    = 32'h40F00000
    = 7.5 ✓
```

**仿真实测输出：** `y_o = 0x40F00000，sign=0, exp=129(0x81), mant=0x700000` ← 完全匹配。

## 流水线时序图

```
时钟周期  │  S1 (拆包/乘/对齐)  │  S2 (加/LOD)  │  S3 (归一化/舍入)  │  输出
─────────┼─────────────────────┼───────────────┼───────────────────┼────────
   N     │ valid_i=1, 锁存输入 │               │                   │
  N+1    │                     │ 处理前一组数据 │                   │
  N+2    │                     │               │ 处理后前一组数据   │
  N+3    │                     │               │                   │ valid_o=1
```

输入从 valid_i 采样边沿算起，结果在第 N+3 个上升沿有效（上表是 N 到 N+3 共 4 拍，实际上是第 3 拍输出——取决于计数方式）。

## 点积模式的仿真实测

```
Dot: Y = Ps + Px×Dx + Py×Dy
    = 1.0 + 2.0 × 1.0 + 3.0 × 1.0
    = 6.0
```

**仿真实测 S1 寄存器值：**

```
s2_exp   = 128                          (锚点指数)
s2_term1 = 40'h2000000000               (Ps 对齐后 = 0.5)
s2_term2 = 40'h4000000000               (Px×Dx = 2.0 的尾数 = 1.0)
s2_term3 = 40'h6000000000               (Py×Dy = 3.0 的尾数 = 1.5)
s2_sign1 = 1, s2_sign2 = 1, s2_sign3 = 1  (全部正)
```

**仿真实测 S2 寄存器值：**

```
s3_mant = 40'hC000000000  (0xC0 = 1100_0000 → bit[39]=1,bit[38]=1 → 2+1 = 3.0)
s3_lod  = 0               (MSB 就在 bit 39)
```

**验算：**
```
Ps_aligned: bit[37]=1 → 2^(-1) = 0.5
Px×Dx:      bit[38]=1 → 2^0 = 1.0
Py×Dy:      bit[38]=1,bit[37]=1 → 1.0 + 0.5 = 1.5
─────────────────────────
sum = 0.5 + 1.0 + 1.5 = 3.0
```

在锚点指数 128 下：3.0 × 2^(128-127) = 3.0 × 2 = **6.0**。

**输出：** `y_o = 0x40C00000, sign=0, exp=129(0x81), mant=0x400000`。
1.5 × 2^(129-127) = 1.5 × 4 = **6.0** ✓
